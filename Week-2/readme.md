SQL Sales Data Analysis – Superstore Dataset

Project Overview

This project analyzes the Superstore Sales Dataset** using **MySQL**. The objective is to perform data exploration, filtering, aggregation, sorting, and business analysis using SQL queries to derive meaningful insights from sales data.

Objectives

* Load the Superstore dataset into a MySQL database.
* Explore the dataset and understand its schema.
* Apply `WHERE` filters to retrieve specific records.
* Perform aggregations using `GROUP BY` functions.
* Identify top-performing products, categories, and customers.
* Analyze monthly sales trends and regional performance.
* Validate data quality by checking row counts, missing values, and duplicate records.

SQL Concepts Used

* `SELECT`
* `WHERE`
* `GROUP BY`
* `ORDER BY`
* `LIMIT`
* Aggregate Functions (`SUM`, `AVG`, `COUNT`)
* `HAVING`
* Data Validation Queries

Business Use Cases Solved

* Sales analysis by region and category.
* Top-selling products and top customers.
* Monthly sales trend analysis.
* Profit analysis by category and region.
* Duplicate record identification.
* Data quality validation.

 Key Insights

* Identified the highest-performing regions and product categories.
* Analyzed customer purchasing patterns and top revenue contributors.
* Detected loss-making orders through profit analysis.
* Evaluated sales trends over time to understand business performance.
* Verified dataset quality by checking duplicates and missing values.

Tools Used

* MySQL Server
* MySQL Workbench
* Superstore Sales Dataset (Kaggle)

 Project Outcome

This project demonstrates practical SQL skills for data analysis, including data exploration, querying, aggregation, and business reporting. It provides valuable insights that can support data-driven business decisions.
